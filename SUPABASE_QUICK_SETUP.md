# 🚀 Supabase 数据库快速设置指南

## 📋 概述

本指南将帮助您在 **5分钟内** 完成 Supabase 数据库设置并添加测试数据。

---

## ✅ 前置条件

在开始之前，请确保您已经：

1. ✅ 拥有一个 Supabase 项目
2. ✅ 在 Supabase Auth 中创建了至少 1 个用户

---

## 🔧 执行步骤

### 步骤 1：创建用户（如果还没有）

1. 访问 Supabase Dashboard：**https://supabase.com/dashboard**
2. 选择您的项目
3. 导航到：**Authentication** → **Users** → **Add user**
4. 填写用户信息：
   - Email: 任意邮箱（如 `admin@example.com`）
   - Password: 任意密码（至少 6 位）
   - ✅ 勾选 **Auto Confirm User**
5. 点击 **Create user**

✅ **用户创建完成！**

---

### 步骤 2：执行数据库设置脚本

1. 在 Supabase Dashboard 中，导航到：**SQL Editor**
   - 直接链接：`https://supabase.com/dashboard/project/YOUR_PROJECT_ID/sql`
   
2. 点击 **New query** 创建新查询

3. 复制 `COMPLETE_DATABASE_SETUP.sql` 文件的**全部内容**并粘贴到 SQL Editor

4. 点击 **Run** (或按 `Ctrl + Enter`)

5. 等待执行完成（约 5-10 秒）

## ❗ 常见问题

### Q1: 执行脚本时报错 "请先创建至少一个用户"

**解决方案**：按照步骤 1 创建用户后再执行脚本。

---

### Q2: 执行脚本时报错 "type already exists"

**解决方案**：这是正常的。脚本使用 `DO` 块处理了已存在的枚举类型，可以忽略此警告。

---

### Q3: 如何重置数据库（删除所有测试数据）？

**解决方案**：在 SQL Editor 中执行以下命令：

```sql
-- 删除所有数据（保留表结构）
TRUNCATE TABLE feeding_records CASCADE;
TRUNCATE TABLE feed_formulas CASCADE;
TRUNCATE TABLE breeding_records CASCADE;
TRUNCATE TABLE milk_records CASCADE;
TRUNCATE TABLE health_records CASCADE;
TRUNCATE TABLE cows CASCADE;

-- 然后重新运行 COMPLETE_DATABASE_SETUP.sql
```

---

### Q4: 可以修改测试数据吗？

**当然可以！** `COMPLETE_DATABASE_SETUP.sql` 中的所有数据都可以根据您的需求修改：
- 奶牛数量、品种、名称
- 健康记录的详细信息
- 产奶量和质量指标
- 饲料配方和成本
