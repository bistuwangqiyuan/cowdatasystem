## 🚀 快速开始

### 方式1: 使用自动化脚本（推荐）

**Windows用户**:
```bash
test-production.bat https://your-site.netlify.app
```

**Linux/Mac用户**:
```bash
chmod +x test-production.sh
./test-production.sh https://your-site.netlify.app
```

### 方式2: 使用命令行

```bash
# 设置生产环境URL
export BASE_URL=https://your-site.netlify.app

# 运行所有生产环境测试
pnpm test:e2e tests/e2e/08-production.spec.ts --config=playwright.config.prod.ts

# 只测试数据分析页面（验证NaN问题）
pnpm test:e2e tests/e2e/08-production.spec.ts -g "数据分析" --config=playwright.config.prod.ts
```

### 方式3: 使用UI模式（最佳调试体验）

```bash
# Windows
set BASE_URL=https://your-site.netlify.app
pnpm test:e2e:ui tests/e2e/08-production.spec.ts --config=playwright.config.prod.ts

# Linux/Mac
BASE_URL=https://your-site.netlify.app pnpm test:e2e:ui tests/e2e/08-production.spec.ts --config=playwright.config.prod.ts
```

## 🎯 关键验证点 - NaN问题

### 重点检查位置:

1. **产奶量卡片**（绿色渐变）:
   ```
   总产奶量
   [数字应该显示，不是NaN]L
   平均: [数字应该显示，不是NaN]L/次
   ```

2. **产奶趋势图**:
   ```
   10/11  25.5 L  ✅
   10/12  30.2 L  ✅
   不应该是: NaN L  ❌
   ```

3. **所有百分比**:
   ```
   异常率: 10.5%  ✅
   成功率: 85.0%  ✅
   不应该是: NaN%  ❌
   ```

### 如果发现NaN:

1. 打开浏览器开发者工具（F12）
2. 查看Console选项卡是否有错误
3. 检查Network选项卡，API请求是否成功
4. 截图并记录具体位置
5. 报告问题

## 🔍 故障排查

### 问题1: 测试提示"页面不可访问"

**解决方案**:
1. 检查URL是否正确
2. 检查网站是否部署成功
3. 在浏览器中手动访问URL验证

### 问题2: 仍然显示NaN

**排查步骤**:
1. 确认生产环境代码是否包含修复
2. 清除浏览器缓存
3. 检查Supabase数据库连接
4. 查看浏览器Console错误信息

### 问题3: 测试超时

**解决方案**:
1. 检查网络连接
2. 增加超时时间（在配置中）
3. 检查Supabase是否响应慢
