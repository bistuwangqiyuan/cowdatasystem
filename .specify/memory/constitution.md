## 项目信息

**项目名称**: 石油用防爆工业磁检测仪器界面软件系统 (Petroleum Explosion-proof Industrial Magnetic Testing Instrument Interface Software System)

**项目简称**: MagSpecKit

**宪章版本**: 1.0.0

**批准日期**: 2025-10-08

**最后修订日期**: 2025-10-08

**项目负责人**: 开发团队

**技术栈**: HTML5 + CSS3 + JavaScript (Vanilla/Modern) + Supabase + Chart.js/ECharts + Tailwind CSS

---

## 项目愿景

打造一个专业级、工业化的石油管道磁检测仪器界面软件系统，符合国际NDT（无损检测）标准（ISO 9934, ASTM E709），提供高精度的磁漏检测、数据采集、实时波形显示和智能分析功能，确保石油管道的安全运行。

---

## 核心原则

本项目遵循以下6项不可妥协的核心原则，所有设计决策、代码实现和运营维护必须严格遵守：

### 原则 1：测试优先开发 (Test-Driven Development)

**描述**:
- 所有功能模块必须在实现前编写单元测试用例
- 测试覆盖率必须达到85%以上
- 关键业务逻辑（数据采集、信号处理、缺陷判定）测试覆盖率必须达到95%
- 每次代码提交前必须通过所有测试
- 使用Jest/Vitest进行单元测试，Playwright进行E2E测试

**理由**: 磁检测系统直接关系到石油管道安全，任何bug都可能导致重大安全事故。测试优先确保代码质量和系统可靠性。

**验收标准**:
- [ ] 每个功能模块有对应的测试文件
- [ ] CI/CD流程包含自动化测试
- [ ] 测试报告显示覆盖率≥85%

---

### 原则 2：代码质量保证 (Code Quality Assurance)

**描述**:
- 使用ESLint + Prettier进行代码规范检查
- 遵循Airbnb JavaScript风格指南
- 所有函数必须有JSDoc注释
- 复杂逻辑必须有行内注释说明
- 单个文件不超过500行
- 单个函数不超过50行
- 代码审查（Code Review）必须由至少一名成员完成
- 圈复杂度不超过10

**理由**: 高质量代码是系统长期可维护性的基础，规范的代码风格降低团队协作成本。

**验收标准**:
- [ ] ESLint检查0错误
- [ ] SonarQube代码质量评级A
- [ ] 所有Pull Request通过Code Review

---

### 原则 4：RESTful API设计 (RESTful API Design)

**描述**:
- 所有数据操作通过Supabase REST API完成
- API设计遵循RESTful规范：
  - GET: 查询数据
  - POST: 创建数据
  - PUT/PATCH: 更新数据
  - DELETE: 删除数据
- 使用语义化的资源命名
- 统一的错误响应格式
- API版本控制（v1, v2...）
- 请求限流和防重放攻击
- 所有API响应时间<500ms

**理由**: 标准化的API设计提升系统可扩展性，便于未来集成第三方系统或移动端应用。

**验收标准**:
- [ ] API文档完整描述所有端点
- [ ] 所有API遵循RESTful命名规范
- [ ] API响应时间符合性能要求

---

### 原则 5：数据安全保护 (Data Security Protection)

**描述**:
- 实施Supabase Row Level Security (RLS)策略
- 用户认证采用JWT Token
- 敏感数据（用户密码、API密钥）必须加密存储
- 数据传输使用HTTPS/TLS 1.3
- 定期数据备份（每日自动备份）
- 审计日志记录所有关键操作（登录、数据修改、配置变更）
- 符合GDPR/CCPA数据隐私要求
- SQL注入、XSS、CSRF攻击防护

**理由**: 磁检测数据是企业核心资产，数据泄露或篡改可能导致法律责任和经济损失。

**验收标准**:
- [ ] Supabase RLS策略配置完成
- [ ] 安全测试通过（OWASP Top 10）
- [ ] 数据备份策略验证有效

---

### 原则 6：完整的监控和日志 (Comprehensive Monitoring and Logging)

**描述**:
必须实现以下监控和日志功能：

**应用层监控**:
- 前端性能监控（页面加载时间、资源加载、用户交互延迟）
- API调用监控（成功率、响应时间、错误率）
- 实时在线用户数

**业务层监控**:
- 检测设备连接状态
- 数据采集频率和质量
- 缺陷识别准确率
- 存储空间使用率

**日志系统**:
- 分级日志：DEBUG, INFO, WARN, ERROR, FATAL
- 结构化日志（JSON格式）
- 日志持久化到Supabase
- 日志保留90天
- 关键错误实时告警（邮件/短信）

**可视化**:
- 实时监控仪表板
- 历史数据趋势图表
- 异常事件时间线

**理由**: 完善的监控和日志是快速定位问题、保障系统稳定运行的关键，符合工业系统运维要求。

**验收标准**:
- [ ] 监控仪表板显示所有关键指标
- [ ] 日志系统正常记录所有级别日志
- [ ] 告警机制有效触发

## 治理规则

### 宪章修订流程

1. 任何团队成员可提出修订提案
2. 提案必须说明修订原因和影响范围
3. 需获得项目负责人和至少2名核心成员批准
4. 版本号按语义化版本规范递增：
   - **主版本号 (MAJOR)**: 删除或重定义核心原则
   - **次版本号 (MINOR)**: 新增原则或重大扩展
   - **修订号 (PATCH)**: 文字修正、格式调整
5. 修订后更新所有相关文档和模板

### 合规性审查

- 每季度进行一次宪章合规性审查
- 每次代码审查时检查是否符合原则要求
- CI/CD流程自动化检查代码质量和测试覆盖率
- 项目上线前进行完整的安全审计

### 违规处理

- 轻微违规：代码退回修改
- 重大违规：触发安全事件响应流程
- 持续违规：重新评估项目架构

## 附录：技术决策记录

### 为什么选择Supabase？

- **原因**: 开源、完整的后端即服务、强大的RLS、实时订阅、易于集成
- **日期**: 2025-10-08

### 为什么不使用框架（React/Vue/Angular）？

- **原因**: 项目强调工业化界面复现，原生HTML/CSS/JS提供最大的UI控制力和性能，避免框架学习成本
- **日期**: 2025-10-08

### 为什么选择Chart.js/ECharts？

- **原因**: 磁信号波形需要高性能图表库，ECharts专业度高，Chart.js轻量级，可按需选择
- **日期**: 2025-10-08
